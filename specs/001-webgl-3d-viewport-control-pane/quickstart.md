# Quickstart: WebGL 3D Viewport + Control Pane Shell

**Branch**: `001-webgl-3d-viewport-control-pane`

---

## Prerequisites

- Node.js ≥ 20 (for pnpm)
- pnpm ≥ 10.12 (`npm i -g pnpm`)
- A Chromium or Firefox browser (WebGL 2 required)

---

## Install & Run

```bash
pnpm install
pnpm dev
```

Open `http://localhost:5173`. You should see:
- Left 2/3: dark 3D viewport with XYZ axis helper (`THREE.AxesHelper`)
- Right 1/3: control pane with a toggle collapse button and a drag handle

> **Note**: The first load may take a few seconds while the OpenCASCADE.js WASM binary
> (~25 MB) is parsed. Subsequent loads are cached by the browser.

## Adding a Workflow Element to the Scene

A workflow element adds a standard `THREE.Object3D` to the scene **and** a
Tweakpane folder to the control pane. It receives both entry points from `main.ts`:

```typescript
import * as THREE from 'three';
import type { SceneManager } from './src/scene/SceneManager.js';
import type { ControlPaneApi } from './src/controls/ControlPane.js';

export function registerFramePart(
  scene: SceneManager,
  controls: ControlPaneApi,
): void {
  const spec = { outerWidth: 100, outerHeight: 80, borderWidth: 5 };

  // — Tweakpane UI —
  const folder = controls.addFolder('Frame Part');
  folder.addBinding(spec, 'outerWidth', { min: 10, max: 500, label: 'Width' });
  folder.addBinding(spec, 'outerHeight', { min: 10, max: 500, label: 'Height' });
  folder.addBinding(spec, 'borderWidth', { min: 1, max: 50, label: 'Border' });

  // — Three.js geometry (OCCT in future) —
  const geometry = new THREE.BoxGeometry(
    spec.outerWidth / 50, spec.outerHeight / 50, 0.1,
  );
  const mesh = new THREE.Mesh(geometry, new THREE.MeshStandardMaterial());
  scene.addObject('frame-part', mesh);

  // Re-generate geometry on parameter change
  folder.on('change', () => {
    geometry.dispose();
    // rebuild mesh from updated spec…
  });
}
```

> **OCCT geometry**: for parametric shapes generated by OpenCASCADE.js, tessellate to
> `THREE.BufferGeometry` first (bridge utility provided in a future feature).

## Architecture Overview

```
main.ts
  ├── THREE.WebGLRenderer (antialias, high-performance GPU hint)
  ├── THREE.PerspectiveCamera (60° FoV) + OrbitControls
  ├── SceneManager(renderer, camera)
  │     └── addObject('axes', new THREE.AxesHelper(2))
  ├── loadOcct() → OccKernel singleton (WASM, async)
  ├── ControlPane(container) → Tweakpane Pane + plugin-essentials registered
  ├── DragHandle → mutates CSS --pane-width + saves LayoutState
  └── RAF loop → SceneManager.renderFrame() each frame
```

---

## Key Design Decisions

- **Three.js** provides scene graph, camera, orbit controls, and geometry — no raw GL shader management needed.
- **OpenCASCADE.js (WASM)** is the parametric geometry kernel; the shell feature only loads it, future workflow elements use it to generate frames, rings, lattices, n-gons, etc.
- **Tweakpane + plugin-essentials** owns the control pane UI; workflow elements inject folders via `ControlPane.addFolder`. No custom HTML form elements needed.
- **SceneRenderer interface** isolates `THREE.WebGLRenderer` (no GL in jsdom) from testable business logic in `SceneManager`. This is the only way to achieve ≥ 98% coverage.
- **Local-first** — all state in `localStorage`; no network calls after initial page load.
- **COOP/COEP headers** (already in `vite.config.ts`) are required for WASM `SharedArrayBuffer` threading.

## Common Issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| Black canvas, no Three.js error | WebGL 2 not supported | Enable hardware acceleration in browser flags |
| Pane width resets on reload | localStorage blocked | Check browser privacy settings (private mode) |
| OCCT load hangs | WASM headers missing | Ensure COOP/COEP headers are sent (check `vite.config.ts`) |
| `COOP`/`COEP` warning in DevTools | Required for WASM threading | Normal — headers set by Vite dev server |
| Coverage < 98% | New code touches `THREE.WebGLRenderer` directly | Wrap behind `SceneRenderer` interface; test via mock |
